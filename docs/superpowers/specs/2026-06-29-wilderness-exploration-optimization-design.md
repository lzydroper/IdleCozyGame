# 荒野探险功能优化设计方案 (Wilderness Exploration Optimization Design)

本设计方案旨在优化地表荒野探索界面（`WildernessTab.tsx`）、遭遇卡牌（`SwipeCard.tsx`）的滑动手感以及现实事件的分类体系，解决原版选项文字局促挤压、滑动手感生硬和事件分类单调的问题。

## 1. 探险卡片与选项布局重构

为了提升遭遇事件的可读性与卡牌感，将采用“卡片与选项分离，双通道交互”的方案。

### 1.1 卡片比例与视觉升级 (`SwipeCard.tsx`)
* **竖向卡牌化**：移除原先的自适应扁平宽度。卡片限定最大宽度 `max-w-[320px]`，并建立 `aspect-[3/4]` 的竖直比例（约 300px * 400px），增强经典的“卡牌/塔罗牌”质感。
* **精美材质**：采用微透毛玻璃（`backdrop-filter: blur(16px)`）和深邃渐变色背景。外框提供不同事件类型对应的流光边框，添加细微的内嵌投影（`box-shadow`）以模拟卡牌的实体厚度。
* **滑动阻尼与回弹**：
  * 在拖拽移动时引入渐进式阻尼公式：`dx_visual = dx * (1 - Math.min(0.5, Math.abs(dx) / 1200))`，使得大位移拖动时手感具有物理分量和张力。
  * 卡片回弹和飞出动画改用更平滑的 `cubic-bezier(0.25, 1, 0.5, 1)` 贝塞尔曲线，带来如丝般顺滑的回弹感。

### 1.2 选项内容解耦与底部决策容器
* **解耦卡片内容**：将事件的详细文本、物资前置消耗与选项名称从卡牌内部抽离，不再塞在卡片底部的一小栏内。
* **决策容器按钮**：在卡片正下方新增两个对称且宽敞的决策按钮（对应选项 A 和选项 B）。
  * 按钮内展示结构化的文本（如“标签图标 + 决策名称”作为主标题，“HP/魔能扣减或所需物品要求”作为副标题）。
  * 支持**点击触发**：不喜欢划动卡牌的玩家可以直接点击按钮做出决定。
  * **滑动联动反馈**：当玩家向左/向右拖拽卡片时，下方的对应按钮会伴随拖动深度平滑高亮（通过过渡背景色与缩放），为玩家提供清晰的心理预期。
* **精简滑动蒙版**：卡片内部的滑动 overlay 不再显示冗长文本，而仅在拖拽时显示精致的指示图标与精简的动作短语（例如 `⬅️ 决定 A` 或 `决定 B ➡️`）。

## 2. 现实事件分类大改动

原本的 4 类（Welfare、Common、Danger、Combat）将全面重构成 **6 大废土事件维度**，以增强现实探索的代入感与策略性。

### 2.1 新分类定义与视觉规范
每个分类都将拥有独特的**代表色**、**卡牌外边框流光**、**类型徽章**，具体规范如下：

1. **废土物资 (Common - 搜刮物资)**
   * *设计定位*：地表残骸的稳定搜集。无直接战斗，通过消耗少量饱食度/体力，稳定搜集废金属、基础材料和低级作物种子。
   * *代表色*：银灰色 (`#a1a1aa`)
   * *类型徽章*：⚙️ 废土物资
2. **生化异变 (Danger - 减Hp/San且产出种子)**
   * *设计定位*：遭遇泄露、生化废墟或辐射沼泽。面临辐射污染，主要损耗 HP，但在深处能采掘到被魔能滋养的高级变异作物种子。
   * *代表色*：荧光绿 (`#22c55e`)
   * *类型徽章*：🧬 生化异变
3. **废土御敌 (Combat - 激烈战斗/要求炮塔)**
   * *设计定位*：变异体、地表土匪或辐射生物的突袭。大幅扣减 HP/魔能，若准备了“防御电磁炮塔”等战斗物资可完美克敌，并能获得变异兽肉等高能食料。
   * *代表色*：猩红色 (`#ef4444`)
   * *类型徽章*：⚔️ 废土御敌
4. **营地休整 (Welfare - 补给/休息)**
   * *设计定位*：地面废弃安全营地、纯净水源或地底温泉。提供安全的避风港，允许玩家打盹以恢复 HP/San，或提取制作能量补给品。
   * *代表色*：治愈金 (`#eab308`)
   * *类型徽章*：🔥 营地休整
5. **古代遗迹 (Relic - 发现梦境碎片/以太)**
   * *设计定位*：旧时代的生化实验室、控制舱或图书馆。极富科技感与神秘色彩，需要消耗魔能解析或破译，可产出梦境碎片、以太和高级合金。
   * *代表色*：神秘紫 (`#a855f7`)
   * *类型徽章*：🔮 古代遗迹
6. **异常天气 (Anomaly - 辐射雨等恶劣环境)**
   * *设计定位*：突发的酸雨风暴、重力紊乱等异常自然伟力。需要消耗粮食躲避或消耗魔能维持护盾，蕴藏伴随天灾而生的奇特物质。
   * *代表色*：橙黄色 (`#f97316`)
   * *类型徽章*：⚡ 异常天气

### 2.2 数据模型适配 (`src/data/realityEvents.ts`)
* 更新 `RealityEventType` 类型：
  ```typescript
  export type RealityEventType = 'common' | 'danger' | 'combat' | 'welfare' | 'relic' | 'anomaly';
  ```
* 将现有的事件适配至新类别。例如：
  * `radiation_leak`（高浓度魔能辐射带） -> `anomaly`（异常天气）
  * `toxic_swamp`（酸雨腐蚀沼泽） -> `danger`（生化异变）
  * `cozy_hotspring`（地底温热泉水） -> `welfare`（营地休整）
  * `ancient_library`（大图书馆遗迹） -> `relic` (古代遗迹)
* 根据每种事件类型重新设计消耗与产出结构，使事件性质更加符合分类描述。
* 在 `WildernessTab.tsx` 中调整类别抽取的随机权重：
  ```typescript
  const CATEGORY_WEIGHTS: Record<string, number> = {
    common: 100,
    anomaly: 80,
    danger: 80,
    combat: 60,
    relic: 50,
    welfare: 40
  };
  ```

## 4. 验证与测试方案

### 4.1 自动化测试
* 运行现有的单元测试和集成测试，确保未破坏基本的探险逻辑：
  `npm run test` 或者 `npx vitest run src/components/WildernessTab.test.tsx`
* 补充对于新类型权重的抽取逻辑单元测试。

### 4.2 手动与视觉验证
* 启动游戏开发服务器 (`npm run dev`)，进入“地表探索”界面。
* 观察遭遇卡牌是否呈现美观的竖直 3:4 比例，以及下方的决策按钮排版是否美观、无重叠。
* 分别测试：
  * **手势/鼠标滑动**：向左右拖拽时，卡牌运动是否带有渐进阻尼，松手飞出或回弹是否自然。
  * **点击操作**：直接点击底部的选项按钮，确认事件能正常结算，且战利品和状态变化正确无误。
* 确认 6 类事件均能按照预期概率被抽取，并在卡片外围展现正确对应的主题色流光。
